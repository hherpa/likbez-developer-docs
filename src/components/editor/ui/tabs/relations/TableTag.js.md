# TableTag Component Documentation

## Описание
TableTag - это компонент для отображения таблицы тегов с количеством использований в интерфейсе редактора. Компонент отображает список тегов с их частотой использования в табличном формате. Таблица сортируется по убыванию количества использований, а при равенстве количества - по алфавиту. Если список тегов пуст, отображается сообщение об отсутствии тегов.

## Экспортируемые компоненты
- `TableTag` - основной компонент таблицы тегов
- `default` - экспорт компонента по умолчанию

## Структура компонента
1. **Контейнер таблицы**:
   - Внешняя граница с закругленными углами
   - Максимальная высота 200px с вертикальной прокруткой

2. **Заголовок таблицы**:
   - Столбец "Тег" с выравниванием по левому краю
   - Столбец "Кол-во" с выравниванием по правому краю и фиксированной шириной 80px
   - Стилизация заголовков с верхним регистром и серым цветом

3. **Тело таблицы**:
   - Строки для каждого тега
   - Название тега в первом столбце
   - Количество использований во втором столбце с табличными цифрами

4. **Пустое состояние**:
   - Сообщение "Теги отсутствуют" при пустом списке
   - Серый цвет текста для лучшей визуальной иерархии

## Взаимодействия
Компонент принимает следующие пропсы:
- `counts` - объект с количеством использований тегов (по умолчанию пустой объект)

Компонент выполняет сортировку тегов:
1. По убыванию количества использований
2. При равенстве количества - по алфавиту (с учетом русской локали)

## Зависимости
- React
- CSS (для стилизации элементов)

## Архитектура
Компонент следует паттерну презентационного компонента React, который отвечает только за отображение данных. Компонент не имеет внутреннего состояния и полностью управляется через пропсы. Используется функциональный подход без хуков, так как компонент не требует управления состоянием. Компонент следует принципу единственной ответственности - он только отвечает за отображение таблицы тегов. Стилизация компонента реализована inline стилями с вынесенными константами для переиспользуемых стилей, что обеспечивает читаемость и поддерживаемость. Компонент экспортируется по умолчанию, что упрощает его импорт в других частях приложения. Сортировка тегов реализована через Object.entries с последующей сортировкой массива. Условный рендеринг пустого состояния реализован через early return, что улучшает читаемость кода. Таблица использует border-collapse для лучшего визуального оформления. Столбец с количеством использует fontVariantNumeric: 'tabular-nums' для выравнивания цифр. Контейнер таблицы имеет ограничение по высоте с вертикальной прокруткой для удобства просмотра большого количества тегов. Компонент не имеет сложных побочных эффектов и зависит только от переданных пропсов.
