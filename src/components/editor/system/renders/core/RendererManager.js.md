# RendererManager.js

## Описание
RendererManager - менеджер рендереров. Соблюдает SOLID принципы:
- SRP: только управление рендерерами
- OCP: легко расширяется новыми рендерерами
- LSP: работает с любыми наследниками BaseRenderer
- ISP: только необходимые методы управления
- DIP: зависит от абстракций (BaseRenderer)

## Экспортируемые компоненты
- `RendererManager` - менеджер рендереров
- `rendererManager` - глобальный экземпляр менеджера рендереров

## Структура
- `constructor()` - инициализация менеджера
- `registerRenderer(elementType, renderer)` - регистрирует рендерер для определенного типа элементов
- `unregisterRenderer(elementType)` - отменяет регистрацию рендерера
- `getRenderer(elementType)` - получает рендерер для определенного типа элементов
- `hasRenderer(elementType)` - проверяет, есть ли рендерер для типа элементов
- `getRegisteredTypes()` - получает список всех зарегистрированных типов элементов
- `setFallbackRenderer(renderer)` - устанавливает fallback рендерер для неизвестных типов
- `renderElement(element, context)` - рендерит элемент используя подходящий рендерер
- `renderElements(elements, context)` - рендерит массив элементов
- `renderUnknownElement(element, context)` - рендерит неизвестный элемент
- `renderError(element, context, error)` - рендерит ошибку
- `clear()` - очищает все зарегистрированные рендереры
- `getStats()` - получает статистику рендереров

## Взаимодействия
- Использует BaseRenderer для регистрации и управления рендерерами
- Использует RenderContext для передачи контекста рендеринга
- Использует React для рендеринга элементов
- Предоставляет глобальный экземпляр rendererManager

## Зависимости
- `react` - для рендеринга React элементов
- `./BaseRenderer` - базовый класс рендерера

## Архитектура
Менеджер рендереров реализует паттерн "Менеджер" для централизованного управления рендерерами:
1. Метод `registerRenderer` позволяет регистрировать рендереры для определенных типов элементов
2. Метод `getRenderer` возвращает подходящий рендерер по типу элемента
3. Метод `renderElement` реализует общий алгоритм рендеринга:
   - Получает подходящий рендерер
   - Обрабатывает ошибки рендеринга
   - Возвращает отрендеренный элемент
4. Метод `renderElements` рендерит массив элементов с автоматической генерацией ключей
5. Поддерживает fallback рендерер для неизвестных типов элементов
6. Предоставляет методы для отладки (renderUnknownElement, renderError) и мониторинга (getStats)

Класс следует принципу инверсии зависимостей, работая с абстракциями (BaseRenderer) вместо конкретных реализаций. Поддерживает как экземпляры BaseRenderer, так и любые объекты, реализующие метод `render(element, context)`.

Глобальный экземпляр `rendererManager` позволяет использовать менеджер рендереров в любом месте приложения без необходимости передачи экземпляра через props.
