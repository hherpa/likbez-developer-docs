# BaseRenderer.js

## Описание
BaseRenderer - базовый класс для всех рендереров. Соблюдает SOLID принципы:
- SRP: только рендеринг одного типа элементов
- OCP: легко расширяется через наследование
- LSP: единый интерфейс для всех рендереров
- ISP: только необходимые методы
- DIP: зависит от абстракций (RenderContext)

## Экспортируемые компоненты
- `RenderContext` - контекст рендеринга
- `BaseRenderer` - базовый класс для всех рендереров
- `RendererFactory` - фабрика для создания рендереров

## Структура
### RenderContext
- `constructor(options)` - инициализация контекста с опциями:
  - `darkMode` - темный режим
  - `theme` - тема
  - `styles` - стили
  - `isInsideReactFlow` - находится ли внутри ReactFlow
  - `nodeProps` - свойства узла
  - `edges` - ребра

### BaseRenderer
- `constructor(elementType)` - конструктор с типом элемента
- `canRender(element)` - проверяет, может ли рендерер обработать элемент
- `render(element, context)` - рендерит элемент
- `isValidElement(element)` - проверяет валидность элемента
- `renderError(element, context)` - рендерит ошибку для невалидного элемента
- `renderElement(element, context)` - абстрактный метод для рендеринга элемента
- `getStyles(context, customStyles)` - получает стили для элемента
- `getThemeColors(context)` - получает цветовую схему для темы

### RendererFactory
- `createRenderer(elementType)` - фабричный метод для создания рендереров

## Взаимодействия
- Использует React для рендеринга элементов
- Зависит от контекста рендеринга (RenderContext)
- Наследуется другими рендерерами для реализации специфичной логики

## Зависимости
- `react` - для рендеринга React элементов

## Архитектура
Базовый класс определяет единый интерфейс для всех рендереров:
1. Метод `canRender` определяет, может ли рендерер обработать элемент
2. Метод `render` реализует общий алгоритм рендеринга:
   - Проверка возможности рендеринга
   - Проверка валидности элемента
   - Рендеринг элемента или ошибки
3. Абстрактный метод `renderElement` должен быть реализован в наследниках
4. Методы `getStyles` и `getThemeColors` обеспечивают темизацию
5. Фабрика `RendererFactory` позволяет создавать рендереры по типу элемента

Контекст рендеринга инкапсулирует все необходимые данные для рендеринга, включая тему, стили и свойства узла.
