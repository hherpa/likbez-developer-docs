# TableLinkedNodes Component Documentation

## Описание
TableLinkedNodes - это компонент для отображения таблицы связанных узлов в интерфейсе редактора. Компонент отображает список связанных узлов в табличном формате с возможностью навигации к узлу или удаления связи. Если список узлов пуст, отображается сообщение об отсутствии связанных узлов.

## Экспортируемые компоненты
- `TableLinkedNodes` - основной компонент таблицы связанных узлов
- `default` - экспорт компонента по умолчанию

## Структура компонента
1. **Заголовок таблицы**:
   - Столбец "Узел" с выравниванием по левому краю
   - Столбец "Действия" с выравниванием по правому краю
   - Стилизация заголовков с верхним регистром и серым цветом

2. **Тело таблицы**:
   - Строки для каждого связанного узла
   - Название узла в первом столбце
   - Кнопки действий во втором столбце:
     - Кнопка "Перейти" для навигации к узлу
     - Кнопка "Удалить" для удаления связи

3. **Пустое состояние**:
   - Сообщение "Связанные узлы отсутствуют" при пустом списке
   - Серый цвет текста для лучшей визуальной иерархии

## Взаимодействия
Компонент принимает следующие пропсы:
- `nodes` - массив связанных узлов (по умолчанию пустой массив)
- `onNavigate` - обработчик навигации к узлу
- `onRemove` - обработчик удаления связи с узлом

Для каждого узла в таблице:
1. Кнопка "Перейти" вызывает обработчик `onNavigate` с объектом узла
2. Кнопка "Удалить" вызывает обработчик `onRemove` с объектом узла

Компонент отображает сообщение об отсутствии узлов, если массив nodes пустой.

## Зависимости
- React
- CSS (для стилизации элементов)

## Архитектура
Компонент следует паттерну презентационного компонента React, который отвечает только за отображение данных и передачу событий родительскому компоненту. Компонент не имеет внутреннего состояния и полностью управляется через пропсы. Используется функциональный подход без хуков, так как компонент не требует управления состоянием. Компонент следует принципу единственной ответственности - он только отвечает за отображение таблицы связанных узлов. Стилизация компонента реализована inline стилями с вынесенными константами для переиспользуемых стилей, что обеспечивает читаемость и поддерживаемость. Компонент экспортируется по умолчанию, что упрощает его импорт в других частях приложения. Все обработчики событий передаются через пропсы, что обеспечивает гибкость в использовании компонента. Условный рендеринг пустого состояния реализован через early return, что улучшает читаемость кода. Таблица использует border-collapse для лучшего визуального оформления. Кнопки действий стилизованы с закругленными углами и границами для улучшения визуального восприятия. Компонент не имеет сложных побочных эффектов и зависит только от переданных пропсов.
