# NodeFooter.js

## Описание
NodeFooter - функция для рендеринга футера узла. Реализует отображение основного тега узла в нижней части узла.

## Экспортируемые компоненты
- `renderFooter(data, parsed, context, config, styles)` - функция для рендеринга футера узла

## Структура
- Функция `renderFooter` принимает данные узла, результат парсинга, контекст, конфигурацию и стили
- Возвращает null если в конфигурации отключен показ футера
- Извлекает основной тег из результата парсинга или данных узла
- Применяет стили футера из конфигурации стилей

## Взаимодействие
- Получает настройку отображения футера из конфигурации
- Извлекает основной тег из результата парсинга или данных узла
- Получает стили футера из стилей
- Возвращает React элемент с отрендеренным футером

## Зависимости
- `react` для создания компонентов

## Архитектура
Функция `renderFooter` реализует рендеринг футера узла, который отображает основной тег узла. Функция следует принципу единственной ответственности, отвечая только за рендеринг футера. Функция использует переданные параметры для получения данных и стилей, что делает ее легко тестируемой и переиспользуемой. Функция поддерживает настройку отображения футера через конфигурацию.

## Принципы SOLID
- Принцип единственной ответственности (SRP): функция отвечает только за рендеринг футера
- Принцип открытости/закрытости (OCP): функция может быть расширена через изменение передаваемых параметров
- Принцип подстановки Барбары Лисков (LSP): функция может быть заменена любой другой реализацией с тем же интерфейсом
- Принцип разделения интерфейса (ISP): функция имеет минимальный необходимый интерфейс
- Принцип инверсии зависимостей (DIP): функция зависит от абстракций (параметров), а не от конкретных реализаций

## Особенности реализации
- Поддержка настройки отображения футера через конфигурацию
- Приоритет извлечения тега из результата парсинга над данными узла
- Использование стилей из переданной конфигурации стилей
- Установка фиксированного размера шрифта и жирности для тега
