# RendererInitializer.js

## Описание
RendererInitializer - инициализирует все рендереры в RendererManager. Соблюдает SOLID принципы:
- SRP: только инициализация рендереров
- OCP: легко добавлять новые рендереры
- LSP: работает с любыми наследниками BaseRenderer
- ISP: только необходимые методы инициализации
- DIP: зависит от абстракций (RendererManager)

## Экспортируемые компоненты
- `initializeRenderers()` - функция для инициализации всех рендереров
- `clearRenderers()` - функция для очистки всех рендереров
- `getRendererStats()` - функция для получения статистики рендереров

## Структура
- Функция `initializeRenderers` регистрирует все рендереры в RendererManager
- Функция `clearRenderers` очищает все рендереры из RendererManager
- Функция `getRendererStats` получает статистику рендереров из RendererManager
- Автоматическая инициализация рендереров при импорте модуля

## Взаимодействие
- Импортирует rendererManager из './RendererManager'
- Импортирует все рендереры из элементов
- Регистрирует рендереры в rendererManager
- Очищает рендереры из rendererManager
- Получает статистику рендереров из rendererManager

## Зависимости
- `./RendererManager` - менеджер рендереров
- Все рендереры из '../elements/'

## Архитектура
Модуль RendererInitializer реализует паттерн "Фабрика" для инициализации рендереров. Модуль следует принципам SOLID:
- Принцип единственной ответственности (SRP): модуль отвечает только за инициализацию рендереров
- Принцип открытости/закрытости (OCP): легко добавлять новые рендереры без изменения существующего кода
- Принцип подстановки Барбары Лисков (LSP): работает с любыми наследниками BaseRenderer
- Принцип разделения интерфейса (ISP): модуль предоставляет только необходимые методы инициализации
- Принцип инверсии зависимостей (DIP): модуль зависит от абстракций (RendererManager)

## Особенности реализации
- Регистрация markdown рендерера для всех типов markdown элементов
- Регистрация LaTeX рендереров для inline и box форматов
- Регистрация рендереров изображений, handles, символов, цветов, тегов и ссылок
- Поддержка автоматической инициализации при импорте модуля
- Поддержка очистки всех рендереров
- Поддержка получения статистики рендереров

- Принцип разделения интерфейса (ISP): модуль предоставляет только необходимые методы инициализации
