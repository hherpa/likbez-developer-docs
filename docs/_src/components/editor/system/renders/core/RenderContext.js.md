# RenderContext.js

## Описание
RenderContext - контекст рендеринга. Содержит все необходимые данные для рендеринга элементов.

## Экспортируемые компоненты
- `RenderContext` - класс контекста рендеринга

## Структура
- Класс `RenderContext` с конструктором и методами
- Метод `createChild` для создания дочернего контекста
- Метод `get` для получения значений из контекста
- Метод `has` для проверки наличия значений в контексте

## Взаимодействие
- Принимает опции в конструкторе для настройки контекста
- Поддерживает иерархию контекстов через родительские ссылки
- Предоставляет методы для работы с данными контекста

## Зависимости
- Нет внешних зависимостей

## Архитектура
Класс `RenderContext` предоставляет контекст для рендеринга элементов, содержащий все необходимые данные. Поддерживает создание дочерних контекстов и предоставляет методы для получения и проверки значений с учетом иерархии контекстов.

## Архитектура
Контекст рендеринга реализует паттерн "Контекст" для передачи данных через иерархию рендеринга:
1. Конструктор инициализирует все необходимые данные для рендеринга
2. Метод `createChild` позволяет создавать дочерние контексты с наследованием свойств
3. Метод `get` обеспечивает доступ к значениям с учетом иерархии (ищет сначала в текущем контексте, затем в родительском)
4. Метод `has` проверяет наличие значения с учетом иерархии

Контекст поддерживает:
- Темизацию (darkMode, theme)
- Идентификацию узла (nodeId, nodeData)
- Иерархию контекстов (parentContext)
- Дополнительные опции рендеринга (renderOptions)

Это позволяет передавать все необходимые данные для рендеринга через единую точку доступа, избегая "проброса пропсов" через всю иерархию компонентов.
