# SearchLinkedNodes Component Documentation

## Описание
SearchLinkedNodes - это компонент для поиска связанных узлов в интерфейсе редактора. Компонент предоставляет поле ввода с автодополнением для поиска узлов по названию. При вводе текста отображается список предложений, из которых пользователь может выбрать нужный узел.

## Экспортируемые компоненты
- `SearchLinkedNodes` - основной компонент поиска связанных узлов
- `default` - экспорт компонента по умолчанию

## Структура компонента
1. **Поле ввода**:
   - input элемент для ввода поискового запроса
   - placeholder с текстом "Поиск узлов..." (настраиваемый)
   - стилизация с закругленными углами и границей

2. **Список предложений**:
   - контейнер со стилизацией (граница, скругленные углы, прокрутка)
   - максимальная высота 160px с вертикальной прокруткой
   - отображается только при наличии результатов поиска

3. **Элементы списка**:
   - div элементы для каждого предложения
   - стилизация с курсором-указателем
   - обработчик клика для выбора элемента

## Взаимодействия
Компонент принимает следующие пропсы:
- `value` - текущее значение поля ввода
- `onChange` - обработчик изменения значения поля ввода
- `suggestions` - массив предложений для автодополнения (по умолчанию пустой массив)
- `onSelect` - обработчик выбора элемента из списка предложений
- `placeholder` - текст-заполнитель для поля ввода (по умолчанию "Поиск узлов...")

Компонент отображает список предложений только когда:
1. Значение поля ввода является строкой
2. Значение поля ввода не пустое (после trim)
3. suggestions является массивом
4. suggestions массив не пустой

При клике на элемент списка вызывается обработчик onSelect с выбранным элементом.

## Зависимости
- React
- CSS (для стилизации элементов)

## Архитектура
Компонент следует паттерну контролируемого компонента React, где состояние поля ввода управляется родительским компонентом. Компонент инкапсулирует логику отображения списка предложений и обработки выбора элемента. Используется функциональный подход без хуков, так как компонент не имеет внутреннего состояния. Компонент следует принципу единственной ответственности - он только отвечает за отображение интерфейса поиска узлов и передачу событий родительскому компоненту. Стилизация компонента реализована inline стилями с вынесенными константами для переиспользуемых стилей, что обеспечивает читаемость и поддерживаемость. Компонент экспортируется по умолчанию, что упрощает его импорт в других частях приложения. Все обработчики событий передаются через пропсы, что обеспечивает гибкость в использовании компонента. Компонент не имеет сложных побочных эффектов и зависит только от переданных пропсов. Условный рендеринг списка предложений реализован через логическое выражение с проверкой всех необходимых условий.
