# ParserConfig.js

## Общее описание
Файл `ParserConfig.js` содержит конфигурационные параметры для парсера редактора. Он определяет паттерны для распознавания различных элементов, цветовую схему, настройки кэширования и другие параметры, используемые в процессе парсинга текста.

## Структура конфигурации

### cache
Настройки кэширования:
- `defaultSize` (number): Размер кэша по умолчанию (100)
- `maxSize` (number): Максимальный размер кэша (1000)

### colors
Цветовая схема:
- Импортируется из `src/components/editor/constants/ColorPalette.js`
- `default` (string): Цвет по умолчанию '#000000'

### supportedColors
Массив поддерживаемых цветов, импортируется из `src/components/editor/constants/ColorPalette.js`

### patterns
Регулярные выражения для парсинга различных элементов:
- `boxLatex`: Паттерн для блочных LaTeX элементов
- `latex`: Паттерн для inline LaTeX элементов
- `colorText`: Паттерн для цветного текста
- `bgColor`: Паттерн для текста с цветом фона
- `image`: Паттерн для изображений
- `handle`: Паттерн для элементов-маркеров
- `symbol`: Паттерн для символьных элементов
- `element`: Паттерн для элементов-тегов
- `link`: Паттерн для ссылок
- `name`: Паттерн для имени элемента

### cleaning
Регулярные выражения для очистки текста:
- `multipleNewlines`: Паттерн для поиска множественных пустых строк
- `replacementNewlines`: Замена для множественных пустых строк

## Использование
Файл используется как централизованное хранилище конфигурации для всех компонентов парсера, обеспечивая согласованность настроек во всем приложении.
