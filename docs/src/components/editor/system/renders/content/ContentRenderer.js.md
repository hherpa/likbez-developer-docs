# ContentRenderer.js

## Описание
ContentRenderer - главный рендерер контента. Только координация рендеринга, делегирует задачи специализированным классам.

## Экспортируемые компоненты
- `ContentRenderer` - главный рендерер контента
- `contentRenderer` - глобальный экземпляр рендерера контента

## Структура
- `constructor()` - инициализация рендерера с ElementBuilder и ElementSorter
- `renderContent(content, context)` - рендерит контент узла
- `parseContent(content)` - парсит контент используя централизованный парсер
- `renderElements(elements, context)` - рендерит массив элементов
- `renderEmptyContent(context)` - рендерит пустой контент
- `renderContentError(content, context, error)` - рендерит ошибку контента
- `getContentContainerStyles(context)` - получает стили контейнера контента
- `renderElement(element, context)` - переопределение для совместимости с BaseRenderer

## Взаимодействия
- Наследуется от BaseRenderer
- Использует ElementBuilder для создания массива элементов
- Использует ElementSorter для сортировки элементов
- Использует rendererManager для рендеринга отдельных элементов
- Использует парсер markdown контента
- Использует React для рендеринга элементов

## Зависимости
- `react` - для рендеринга React элементов
- `../core/BaseRenderer` - базовый класс рендерера
- `../core/RenderContext` - контекст рендеринга
- `../core/RendererManager` - менеджер рендереров
- `../../parser` - парсер markdown
- `./ElementBuilder` - строитель элементов
- `./ElementSorter` - сортировщик элементов

## Архитектура
Рендерер контента реализует паттерн "Фасад" для координации рендеринга:
1. Метод `renderContent` реализует общий алгоритм рендеринга контента:
   - Парсит контент
   - Создает массив элементов
   - Сортирует элементы
   - Рендерит элементы
2. Делегирует задачи специализированным классам:
   - ElementBuilder для создания массива элементов
   - ElementSorter для сортировки элементов
   - rendererManager для рендеринга отдельных элементов
3. Обрабатывает ошибки рендеринга и пустой контент
4. Предоставляет методы для рендеринга отдельных частей (пустой контент, ошибки)

Класс следует принципу единственной ответственности, координируя процесс рендеринга, но не реализуя логику рендеринга отдельных элементов. Использует rendererManager для делегирования рендеринга отдельным рендерерам.

Глобальный экземпляр `contentRenderer` позволяет использовать рендерер контента в любом месте приложения без необходимости передачи экземпляра через props.
